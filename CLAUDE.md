This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Real-time traffic simulation dashboard using SUMO (Simulation of Urban MObility). A 4×3 grid of 12 intersections is simulated and monitored in real time. Everything runs in Docker.

## Commands

### Build & Run

```bash
# Build and start everything
docker compose up --build

# Build a specific service only
docker compose build frontend
docker compose build spring-backend
docker compose build sumo-adapter

# Rebuild and restart a single service
docker compose up -d --build frontend

# View logs
docker compose logs -f sumo-adapter
docker compose logs -f spring-backend
```

### Frontend development (outside Docker)

```bash
cd frontend
npm ci
ng serve          # dev server on http://localhost:4200
ng build          # production build
ng test           # run tests
```

### Backend development (outside Docker)

```bash
cd backend
mvn clean package -DskipTests
mvn spring-boot:run   # requires a running PostgreSQL on localhost:5432
```

### SUMO adapter (outside Docker)

```bash
pip3 install aiohttp websockets
SUMO_HOME=/usr/share/sumo python3 controle_Traci.py
```

## Architecture

### Service map

```
[Angular frontend :8080]
    ↕ HTTP  → GET /api/* → [Spring Boot :8081]
    ↕ WS    → /ws (STOMP) → [Spring Boot]

[Spring Boot :8081]
    ↕ PostgreSQL → persist TrafficSnapshotEntity
    ↕ HTTP POST http://sumo-adapter:8766/command → SUMO adapter

[SUMO adapter :8765/:8766]
    ↕ HTTP POST http://spring-backend:8080/api/traffic/snapshot → Spring Boot
    ↕ TraCI → SUMO simulation engine (sumo binary, headless in Docker)
```

### Data flow (snapshot cycle)

Every simulation step, `controle_Traci.py` collects vehicle positions and traffic light states, then:
1. POSTs JSON to `POST /api/traffic/snapshot`
2. `TrafficIngestionService` deserialises → saves to `traffic_snapshot` table → calls `TrafficWsPublisher`
3. `TrafficWsPublisher` sends to STOMP topic `/topic/traffic`
4. `TrafficWsService` (Angular) receives, stores in signals `lastMessage` and `snapshotsByTlsId` (keyed by `tlsId`)
5. `NetworkPage` reads signals in its `requestAnimationFrame` loop and renders to HTML5 Canvas

### Command flow (user action)

`ShellPage` → `SimulationService.start/stop/setScenario()` → `fetch('/api/simulation/*')` → `SimulationController` → `SimulationCommandService.sendCommand()` → `POST http://sumo-adapter:8766/command` → `controle_Traci.py` queues the action for the next simulation step.

All simulation control endpoints return `202 Accepted` immediately (fire-and-forget); the SUMO adapter processes commands asynchronously.

### Key backend classes

| Class | Role |
|---|---|
| `TrafficIngestionService` | Receives raw JSON snapshot, persists + publishes WebSocket |
| `TrafficWsPublisher` | Single point that calls `SimpMessagingTemplate.convertAndSend("/topic/traffic", ...)` |
| `SimulationCommandService` | Sends typed commands to SUMO adapter via `RestTemplate` |
| `SimulationController` | REST endpoints, supports both JSON body and query-param variants |
| `AppConfig` | Declares `RestTemplate` and `ObjectMapper` Spring beans |
| `SecurityConfig` | CSRF disabled, CORS `allowedOriginPatterns("*")`, all requests permitted (no auth) |
| `WebSocketConfig` | STOMP over SockJS at `/ws`, origin `*`, broker on `/topic`, app prefix `/app` |

### Key frontend services/components

| File | Role |
|---|---|
| `TrafficWsService` | STOMP client; exposes `connected`, `lastMessage`, `snapshotsByTlsId` as signals |
| `SimulationService` | REST client for all `/api/simulation/*` endpoints; tracks active incidents in a signal |
| `ShellPage` | App shell; owns simulation controls (scenario, speed, incidents) and WebSocket lifecycle |
| `NetworkPage` | HTML5 Canvas renderer; runs a `requestAnimationFrame` loop; parses `reseau_12_carrefours.net.xml` for road geometry |

### SUMO adapter internals (`controle_Traci.py`)

Three concurrent asyncio tasks:
- **`simulation_loop`**: TraCI step loop; applies queued commands, collects data, POSTs snapshot to Spring
- **`http_control_loop`**: aiohttp server on `:8766`; accepts `POST /command` and enqueues actions
- **`websocket_broadcast_loop`**: legacy WebSocket broadcast on `:8765` (kept for compatibility)

Incident types handled by TraCI: `ACCIDENT` (speed ×5%), `BREAKDOWN` (speed ×10%), `TRAVAUX` (speed ×1% + lane closed).

### SUMO network

- File: `reseau_12_carrefours.net.xml` (4×3 grid, junction IDs: A0–D2)
- 2 lanes per direction = 4 lanes per road segment
- Served statically by Nginx so the Angular frontend can parse it at runtime (`fetch('/reseau_12_carrefours.net.xml')`)
- Regenerated by `generer_simulation.py` when scenario changes; traffic files: `trafic.rou.xml`, `trips.trips.xml`

## Important constraints

- **No authentication**: Keycloak was removed. All endpoints are open. `AuthService` is a no-op stub.
- **Stateless backend**: sessions are `STATELESS`; no session cookies.
- **Single WebSocket publish path**: only `TrafficWsPublisher` should call `convertAndSend("/topic/traffic", ...)`. Do not add a second publish point in `TrafficIngestionService` or elsewhere.
- **`RestTemplate` and `ObjectMapper` are Spring beans** declared in `AppConfig`; inject them, never instantiate manually.
- **`ddl-auto: update`**: schema is auto-managed by Hibernate; do not write manual migration scripts unless switching to `validate`.
- **Canvas rendering is geometry-sensitive**: `NetworkPage.draw()` uses `requestAnimationFrame` with Angular signals (not `OnPush` change detection). Keep heavy computation out of the animation loop; precompute in `loadNet()`/`parseNet()`.
